Welcome to **GpgFrontend**, a powerful, cross-platform [OpenPGP](https://www.openpgp.org/) tool designed to make encryption accessible for everyone. This guide will help you set up your environment and perform your first secure operations.

<Aside type="caution" title="Important: GnuPG Version Requirement">
  Starting from version 2.1.8, GpgFrontend requires **GnuPG 2.4 or later** for
  full feature support. While GnuPG 2.2 is the minimum to start, some advanced
  functions may be disabled.
</Aside>

## Cryptographic Engines

GpgFrontend relies on a backend engine to handle cryptographic tasks. Refer to the table below to see what your system needs:

| Platform         | Recommended Engine  | Notes                                                      |
| :--------------- | :------------------ | :--------------------------------------------------------- |
| **Windows**      | **GnuPG** (Bundled) | No separate installation required.                         |
| **macOS**        | **GnuPG**           | Install via Homebrew: `brew install gnupg`.                |
| **macOS (Lite)** | **rPGP** (Built-in) | App Store version; runs in a sandbox.                      |
| **Linux**        | **GnuPG**           | Pre-installed in most distros. Check with `gpg --version`. |
| **BSD**          | **GnuPG**           | Must be installed manually via ports/pkg.                  |

> **Note:** If GnuPG is missing (v2.2.0+), GpgFrontend will automatically fall back to the built-in **rPGP** engine to maintain basic functionality.

## Installation Guide

Select your operating system to view the recommended installation steps. For more advanced methods (WinGet, Flatpak), visit our [Downloads](/overview/downloads) page.

<Tabs>
  <TabItem label="Windows">
    ### Installer (Recommended)
    <Steps>
    1. **Download** the `.msix` installer from [GitHub Releases](https://github.com/saturneric/GpgFrontend/releases/latest).
    2. **Launch** the file and follow the on-screen prompts.
    3. **Open** GpgFrontend from your Start Menu.
    </Steps>

    ### Portable Version
    <Steps>
    1. **Download** the `windows-x86_64-portable.zip`.
    2. **Extract** the ZIP to your preferred folder.
    3. **Run** `GpgFrontend.exe` located in the `bin` directory.
    </Steps>

  </TabItem>

<TabItem label="macOS">
  <Steps>
    1. **Install GnuPG** via Homebrew: `brew install gnupg`. 2. **Download** the
    `.dmg` file from [GitHub
    Releases](https://github.com/saturneric/GpgFrontend/releases/latest). 3.
    **Mount** the DMG and drag **GpgFrontend** to your **Applications** folder.
  </Steps>
  ![Drag to
  Applications](https://image.cdn.bktus.com/i/2025/06/24/03df3c64a965b9da96c82ddb38c35b74b1796ee0.webp)
</TabItem>

<TabItem label="Linux">
  <Steps>
    1. **Verify GnuPG**: Ensure `gpg` is installed (`sudo apt install gpg` or
    `sudo dnf install gnupg`). 2. **Download** the `.AppImage` from [GitHub
    Releases](https://github.com/saturneric/GpgFrontend/releases/latest). 3.
    **Set Permissions**: Run `chmod u+x ./GpgFrontend-xxx.AppImage`. 4.
    **Launch**: Double-click the file or run it via terminal.
  </Steps>
</TabItem>

  <TabItem label="BSD">
    Currently, GpgFrontend must be **compiled from source** on BSD systems. Please refer to our [Build Instructions](https://github.com/saturneric/GpgFrontend) on GitHub for a step-by-step guide.
  </TabItem>
</Tabs>
