### Artifact Nomenclature

Below is the decoding table for our release artifacts. Use this to identify the exact package engineered for your operating environment.

| Filename Pattern               | Operational Purpose                                                   |
| ------------------------------ | --------------------------------------------------------------------- |
| `-windows-x86_64-portable.zip` | Zero-footprint Windows execution. Run directly from a USB drive.      |
| `-windows-x86_64.msix`         | Modern MSIX deployment for Windows (Recommended).                     |
| `-windows-x86_64.msi`          | Standard MSI installer; engineered for Enterprise environments.       |
| `-macos-<ver>.dmg`             | macOS disk image; `<ver>` indicates minimum OS requirement.           |
| `-linux-<arch>.AppImage`       | Standalone Linux executable. Grant execution rights and ignite.       |
| `*.sig`                        | Cryptographic signature. Always verify the integrity of your machine. |
| `-qt5-`                        | Legacy architecture build (e.g., Windows 7, Ubuntu 20.04).            |

:::note[The Lite Architecture]

**GpgFrontend Lite** is specifically engineered for the strict macOS sandbox
environment. By omitting GnuPG and relying exclusively on the memory-safe **rPGP
engine**, it delivers a seamless Mac App Store experience.

For the complete cryptographic workstation—including GnuPG legacy support and
advanced hardware token integration—please deploy the standard Desktop Edition
via Homebrew or GitHub.

:::

:::caution[Operational Directive: Secure Your Keys]

If you operate **GpgFrontend Lite**, immediate backup of your OpenPGP keys is
mandatory. Because the Lite edition stores cryptographic material within an
ephemeral Apple sandbox, application removal or system resets may obliterate
your keys. **Lost keys result in permanently inaccessible data.**

:::
