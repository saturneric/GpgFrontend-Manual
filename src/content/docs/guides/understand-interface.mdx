## The Window at a Glance

The window has four main parts:

- **Operations Bar** (top): the main buttons, like Encrypt, Decrypt, Sign, and
  Verify.
- **Workspaces** (center): tabs where you work on files, text, or email.
- **Key Toolbox** (right): the list of your keys.
- **Status Panel**: shows the result of each task.

The rest of this page looks at each part in turn.

## Workspaces

GpgFrontend splits your work into tabs called workspaces, so you can work on
files, plain text, and email at the same time.

There are three kinds of workspace, each shown as a tab:

- **File Panel**: work on files and folders.
- **Text Editor**: write, encrypt, and decrypt plain text.
- **Mail Editor**: work with email messages.

You can open many tabs of each kind and switch between them whenever you like.

### File Panel

The File Panel handles tasks on files and folders. You can:

- Browse and select files or folders from your system.
- Encrypt, decrypt, sign, or verify one file or many at once.
- Choose between text (ASCII) and binary output.
- Turn on Batch Mode to work on many files at once.
- Automatically pack and encrypt whole folders (see the File Operations guide for
  details).

Files and folders you open always appear in a File Panel tab.

### Text Editor

The Text Editor lets you type new text or open existing files.

#### Main Features

- **Operations Bar**: at the top, with buttons to encrypt, decrypt, sign, and
  verify text.
- **Standard Shortcuts**: supports basic shortcuts like **Ctrl+C**, **Ctrl+V**,
  and **Ctrl+S** for copy, paste, and save.
- **Plain Text Only**: text is saved as UTF-8 with no formatting, for clarity and
  compatibility. Rich text editing may come later.
- Files up to **1 MB** work best. Bigger files may slow things down and aren't
  recommended.
- While a large file loads, editing is paused until it finishes.

### Mail Editor

The Mail Editor is made for email messages in EML format.

- **Open and edit EML files**: load .eml files (a standard email format) to view
  and edit them.
- **Write and secure email**: write or change email content, then encrypt, sign,
  or verify it.

:::caution[Note]

The Mail Editor works best with .eml files. This keeps things compatible with
normal email apps and keeps the formatting and details intact during your tasks.

:::

## Status Panel

The **Status Panel** shows what happened after each task (encrypt, decrypt,
sign) and whether it worked. It also shows useful details like the time, key IDs,
and algorithms used.

### Color Coding for Status

- **Green**: the task worked, no problems.
- **Yellow**: the task worked, but with a warning to check.
- **Red**: the task failed, or there was a serious error.

### Switching Views

The panel has two views. Use the switch at the top to change them:

- **Status**: a short, easy-to-read summary of the task.
- **Details**: the full, raw text straight from the engine.

Your choice is remembered, so the panel keeps showing the view you like.

### Panel Actions

- **Copy**: copies the status text to the clipboard.
- **Save**: saves the status text as a plain text file (UTF-8).
- **Magnify**: opens the summary in a bigger view. Available only when the
  **Status** view is showing a summary.
- **Clear**: empties the panel, ready for the next task.

## Key Toolbox

On the right side, the **Key Toolbox** lists all your keys: public keys, private
keys, and key groups.

Main features:

- **Selection**: tick the boxes to pick keys for a task.
- **Type / Usage / Email**: see at a glance what each key is for.
- **Tab filtering**: use the tabs to filter keys by type or status.

Keys that are expired, revoked, or unavailable are hidden by default. You can find
them in the **Manage Keys** window.

### Reading the Key Toolbox Table

Each key has some columns and short codes (in the Type and Usage columns) that
aren't obvious at first. Here is what they mean.

- **Type**:
  - `pub`: a public key (for encryption or verification).
  - `pub/sec`: a key pair with both a public and a private key.
  - `pub/sec#`: a key pair whose primary key is missing.
  - `pub/sec^`: a key pair stored on a smart card.
  - `group`: a key group (several public keys used together as one recipient).
- **Usage** (what the key can do):
  - `C` (Certification): can sign other keys (usually primary keys only).
  - `E` (Encryption): can encrypt data to this key.
  - `S` (Signing): can create digital signatures.
  - `A` (Authentication): can be used to log in (for example, SSH).

## Operations Bar

The **Operations Bar** at the top gives you one-click access to the main
features. Here are the buttons, from left to right:

1. **Open**: open a file or folder in the File Panel.
2. **Workspace**: manage and switch between tabs (File Panel, Text Editor, Mail
   Editor).
3. **Encrypt**: lock the selected files or text for the chosen recipients.
4. **Decrypt**: unlock the selected files or text. You don't need to pick a key;
   GPG finds the right private key for you.
5. **Sign**: sign the selected files or text with your private key.
6. **Verify**: check the signature on the selected files or text.
7. **New Keypair**: make a new key pair.
8. **Import Key...**: add existing public or private keys.
9. **Manage Keys**: open the key management window.

### Customization

You can hide buttons you don't use, or add ones you use often, from the view
settings menu.
